## 二、实验原理

### 2.1 质点-弹簧模型

将布料离散化为 $N \times N$ 的质点网格，质点之间通过弹簧相连。

**弹簧力（胡克定律）：**

$$f_{a} = -k_{s} \left(|x_a - x_b| - l\right) \frac{x_a - x_b}{|x_a - x_b|}$$

其中 $k_s$ 为劲度系数，$l$ 为弹簧原长，$x$ 为质点位置。力的方向沿弹簧轴线，大小与形变量成正比。

**阻尼力：**

$$f_{d} = -k_{d} v_{a}$$

阻尼力与速度反向，用于耗散系统能量，防止数值积分过程中能量无限累积导致发散。

**合力与加速度（牛顿第二定律）：**

$$F = f_{spring} + f_{damping} + f_{gravity}, \quad a = F / m$$

### 2.2 弹簧拓扑结构（必做 + 选做）

本实验实现三类弹簧，覆盖布料抵抗不同形变的物理约束：

| 类型 | 连接方式 | 约束的形变 | 劲度系数 |
|------|----------|-----------|---------|
| 结构弹簧（Structural） | 横向/纵向相邻质点 | 拉伸与压缩 | $k_s = 10000$ |
| 剪切弹簧（Shear，选做） | 对角线相邻质点 | 剪切变形 | $k_{shear} = 5000$ |
| 弯曲弹簧（Bending，选做） | 间隔一个质点的横向/纵向 | 弯曲刚度 | $k_{bend} = 2000$ |

三类弹簧的拓扑示意（以 4×4 网格中某质点为例）：

```

O - S - O - S - O       S: 结构弹簧
|\ B \  |\ B \  |       X: 剪切弹簧（对角）
S  X  S  X  S  S       B: 弯曲弹簧（间隔1）
|    \ | /    \ |
O - S - O - S - O

```

### 2.3 数值积分方法

设当前时刻状态为 $(x_t, v_t)$，时间步长为 $\Delta t$，加速度 $a_t = F(x_t, v_t)/m$。

**显式欧拉（Explicit Euler）：**

$$x_{t+1} = x_t + v_t \Delta t$$
$$v_{t+1} = v_t + a_t \Delta t$$

完全用当前状态预测下一状态。实现最简单，但对 $\Delta t$ 极度敏感，容易数值发散。

**半隐式欧拉（Semi-Implicit Euler）：**

$$v_{t+1} = v_t + a_t \Delta t$$
$$x_{t+1} = x_t + v_{t+1} \Delta t$$

先更新速度再用新速度更新位置。这一微小的顺序调整使其具备辛积分器（Symplectic Integrator）的能量守恒特性，稳定性显著优于显式欧拉，是实时物理模拟中最常用的方案。

**隐式欧拉（Implicit Euler，定点迭代近似）：**

$$v_{t+1} = v_t + a(x_{t+1}, v_{t+1}) \Delta t$$
$$x_{t+1} = x_t + v_{t+1} \Delta t$$

理想情况下需要求解非线性方程组。本实验使用**定点迭代法**近似：以当前状态为初值，反复用预测状态重新计算受力并更新，迭代 3 次即可获得较好近似。最稳定，允许使用更大时间步长。

### 2.4 数值爆炸与速度钳制

显式欧拉在 $\Delta t$ 过大或弹簧劲度系数过高时，质点速度会在一步内指数级增长（数值爆炸）。引入最大速度限制：

$$v \leftarrow \begin{cases} v & \|v\| \leq v_{max} \\ v_{max} \cdot \hat{v} & \|v\| > v_{max} \end{cases}$$

这是一种工程妥协，以轻微的物理不精确换取数值稳定性。

### 2.5 球体碰撞（选做）

对每个非固定质点，判断其与球心的距离：

$$d = \|x_i - c\|$$

若 $d < r_{sphere}$，发生穿透，按以下步骤处理：

1. **位置修正**：沿法线方向将质点推出球面：$x_i \leftarrow x_i + \hat{n} \cdot (r - d)$
2. **速度反弹**：消除沿法线方向的穿入速度分量，引入恢复系数 $e = 0.3$：

$$v_i \leftarrow v_i - (1 + e)(v_i \cdot \hat{n})\hat{n}, \quad \text{当} \; v_i \cdot \hat{n} < 0$$

| 弯曲弹簧 $k_{bend}$ | 2000 | 弯曲劲度系数（选做） |

### 4.2 初始化架构设计

实验要求将初始化拆分为多个 `@ti.kernel`，分别处理位置、弹簧拓扑和渲染索引，在 Python 侧按顺序调用，以保证多线程下 GPU 计算的状态同步：

```python
def init_cloth():
    num_springs[None] = 0
    init_positions()       # Kernel 1：初始化质点位置、速度、固定点
    init_springs()         # Kernel 2：计算弹簧拓扑与原长
    init_spring_indices()  # Kernel 3：生成渲染用的线段索引数组
````

**为什么不合并成一个 Kernel？**

`init_springs()` 中需要读取 `x[idx]` 计算弹簧原长，而 `x` 由 `init_positions()` 写入。若合并在同一个 Kernel 内，Taichi 的并行执行无法保证写入顺序，可能读到未初始化的位置数据。拆分后，Python 侧的顺序调用在每两个 Kernel 之间隐式插入 GPU 同步屏障，保证数据依赖的正确性。

### 4.3 固定点设置

布料顶边的两个角点（`j==0` 且 `i==0` 或 `i==N-1`）被设为固定点，模拟布料被悬挂的效果：

```python
if j == 0 and (i == 0 or i == N - 1):
    is_fixed[idx] = 1
```

## 五、核心代码说明

### 5.1 ti.func 与 ti.kernel 的架构选择

| 特性   | `@ti.kernel`     | `@ti.func`       |
| ---- | ---------------- | ---------------- |
| 调用方  | Python 侧         | 其他 Kernel 或 Func |
| 编译行为 | 独立编译为 GPU Kernel | 强制内联到调用方         |
| 并行范围 | 最外层 for 自动并行     | 继承调用方的并行上下文      |
| 启动开销 | 每次调用有开销          | 无额外调用开销          |
| 使用场景 | 每帧调用的顶层操作        | 力计算、速度钳制等复用逻辑    |

`compute_forces_on()` 和 `clamp_velocity()` 声明为 `@ti.func`，在编译时内联到各积分 Kernel 中，避免重复的 Kernel 启动开销。

### 5.2 弹力计算与原子操作

弹簧力是成对作用的：弹簧 $(a, b)$ 对质点 $a$ 施加一个力，同时对质点 $b$ 施加等大反向的力。在 GPU 并行环境下，多个线程可能同时向同一质点累加力，必须用 `ti.atomic_add` 避免写入竞争：

```python
@ti.func
def compute_forces_on(pos: ti.template(), vel: ti.template(), force: ti.template()):
    # 重力 + 阻尼（每个质点独立，无竞争）
    for i in range(N * N):
        force[i] = gravity * mass - k_d * vel[i]

    # 弹簧力（两端共享，需原子操作）
    for i in range(num_springs[None]):
        idx_a = spring_pairs[i][0]
        idx_b = spring_pairs[i][1]
        d = pos[idx_a] - pos[idx_b]
        dist = d.norm()
        if dist > 1e-6:
            f_spring = -k_s * (dist - spring_lengths[i]) * (d / dist)
            ti.atomic_add(force[idx_a],  f_spring)
            ti.atomic_add(force[idx_b], -f_spring)
```

### 5.3 三种积分器实现

**显式欧拉：**

```python
@ti.kernel
def step_explicit():
    compute_forces_on(x, v, f)
    for i in range(N * N):
        if is_fixed[i] == 0:
            x[i] += v[i] * dt          # 用旧速度更新位置
            v[i] += (f[i] / mass) * dt  # 用旧受力更新速度
            clamp_velocity(v, i)
```

**半隐式欧拉：**

```python
@ti.kernel
def step_semi_implicit():
    compute_forces_on(x, v, f)
    for i in range(N * N):
        if is_fixed[i] == 0:
            v[i] += (f[i] / mass) * dt  # 先更新速度
            clamp_velocity(v, i)
            x[i] += v[i] * dt           # 用新速度更新位置
```

**隐式欧拉（3 次定点迭代）：**

```python
@ti.kernel
def step_implicit_iter():
    # 以当前状态为初始猜测
    for i in range(N * N):
        v_next[i] = v[i]
        x_next[i] = x[i]
    # 定点迭代 3 次
    for _ in ti.static(range(3)):
        compute_forces_on(x_next, v_next, f_next)
        for i in range(N * N):
            if is_fixed[i] == 0:
                v_next[i] = v[i] + (f_next[i] / mass) * dt
                clamp_velocity(v_next, i)
                x_next[i] = x[i] + v_next[i] * dt
    # 将迭代结果写回
    for i in range(N * N):
        v[i] = v_next[i]
        x[i] = x_next[i]
```

`ti.static(range(3))` 让编译器在编译期展开循环，避免运行时的循环控制开销，是 Taichi 中对固定次数迭代的标准优化写法。

### 5.4 球体碰撞（选做）

碰撞检测与响应嵌入在 `compute_forces_on()` 内，与力计算同步执行：

```python
center = sphere_center[0]
for i in range(N * N):
    if is_fixed[i] == 0:
        p = pos[i]
        dir_to_center = p - center
        dist = dir_to_center.norm()
        if dist < sphere_radius and dist > 1e-8:
            normal = dir_to_center / dist
            penetration = sphere_radius - dist
            pos[i] += normal * penetration   # 位置修正，推出球面
            vn = vel[i].dot(normal)
            if vn < 0:                        # 只处理穿入方向的速度
                vel[i] -= (1 + 0.3) * vn * normal  # 弹性反弹，e=0.3
```

### 6.1 必做：三种积分方法对比（阻尼 = 1.0）

**稳定性分析：**

| 积分方法  | 稳定性        | 能量行为        | 适用场景       |
| ----- | ---------- | ----------- | ---------- |
| 显式欧拉  | 差（条件稳定）    | 能量随时间增加，易发散 | 仅用于学习对比    |
| 半隐式欧拉 | 好（无条件稳定趋势） | 能量守恒，长期稳定   | 实时物理模拟首选   |
| 隐式欧拉  | 最好         | 能量耗散，过阻尼    | 允许大时间步长的场景 |

* **显式欧拉**：阻尼较小或时间步长稍大时，布料会出现抖动甚至爆炸，需要速度钳制介入才能维持稳定，物理真实性最差；
* **半隐式欧拉**：速度先于位置更新，使系统具备辛积分特性，在相同参数下明显更稳定，布料运动自然流畅；
* **隐式欧拉**：定点迭代近似使系统偏向过阻尼，运动略显"粘滞"，但在大时间步长下依然稳定，不会发散。

## 七、参数分析

### 7.1 阻尼系数的影响

| 阻尼 $k_d$ | 现象                    |
| -------- | --------------------- |
| 0.0      | 系统无耗散，布料持续振荡不停歇       |
| 1.0（本实验） | 适度耗散，布料经过若干次振荡后趋于静止   |
| 5.0      | 高耗散，布料几乎不振荡，快速静止      |
| 过大       | 阻尼力过强，质点几乎无法运动，布料"冻结" |

### 7.2 弹簧劲度系数的影响

$k_s$ 越大，弹簧越"硬"，布料形变越小，但同时对数值积分的稳定性要求越高（需要更小的 $\Delta t$ 或更稳定的积分方法）。本实验使用 $k_s = 10000$，在半隐式和隐式欧拉下稳定，在显式欧拉下需要速度钳制辅助。

## 八、GPU 编程要点总结

| 要点              | 说明                                         |
| --------------- | ------------------------------------------ |
| Kernel 拆分       | 有数据依赖的操作必须拆分为多个 Kernel，依靠 Python 侧顺序调用插入同步 |
| `ti.atomic_add` | 多线程对同一内存位置累加时的唯一正确写法，防止写入竞争                |
| `ti.func` 内联    | 复用逻辑封装为 `@ti.func`，编译时内联，避免重复 Kernel 启动开销  |
| `ti.static`     | 编译期展开固定次数的循环，消除运行时循环控制开销                   |
| `ti.template()` | 允许 `@ti.func` 接受 Taichi field 作为参数，实现泛型复用  |

## 九、实验总结

1. **积分方法的稳定性本质**：显式欧拉用"过去"预测"未来"，在弹簧力较大时误差逐步放大，系统能量单调递增直至发散。半隐式欧拉只是调换了速度和位置的更新顺序，却使系统变为辛积分器，从根本上改变了能量行为——这是数值方法中"顺序决定稳定性"的经典案例。

2. **正则化 vs 速度钳制**：速度钳制是一种强制截断，以破坏物理守恒为代价换取稳定性。在生产级模拟中，更合理的做法是选用稳定的积分方法（半隐式或隐式欧拉），而非依赖钳制来"救场"。

3. **三类弹簧的物理意义**：结构弹簧抵抗拉伸，剪切弹簧抵抗斜向变形，弯曲弹簧抵抗折叠——三者共同构成布料各向异性刚度的离散近似。缺少任一类都会导致布料在某个自由度上表现失真。

4. **GPU 并行的编程范式**：Taichi 的 Kernel/Func 分层设计与 CUDA 的 Kernel/Device Function 高度对应。`ti.atomic_add` 解决的写入竞争问题、Kernel 拆分解决的数据依赖问题，在任何 GPU 并行编程框架中都是必须面对的基础挑战，本实验提供了处理这类问题的完整范例。

5. **碰撞检测的简化与局限**：本实验的球体碰撞采用逐质点距离判断，时间复杂度 $O(N^2)$，对于小规模布料已足够。真实场景中的布料-物体碰撞需要层次包围盒（BVH）等空间加速结构，以及连续碰撞检测（CCD）来处理高速穿透问题，是计算机图形学中持续活跃的研究方向。
